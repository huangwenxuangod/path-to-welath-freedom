技术栈：TypeScript + TSX + React + Next.js (最新版 App Router, SSR) + Hero UI + Tailwind CSS (默认配置)

包管理器：Bun
代码格式化：Biome.js
部署平台：Vercel

状态管理：Zustand (全局状态) + TanStack Query (数据获取与缓存)
表单处理：React Hook Form + Zod

后端服务：
- Supabase (数据库)
- Clerk (Auth 认证)
- Resend (邮件营销管理)
- Stripe (海外支付：一次性、月费、年费订阅)
- UploadThing (文件上传)
- RevenueCat (收益分析)
- PostHog (数据分析)

API 路由：Next.js Route Handlers (与 Supabase Edge Functions 可共同使用)

测试：Vitest

动画：Framer Motion
图标：Lucide React

移动端：React Native + Expo Go + NativeWind + Lucide React
浏览器插件：WXT + React + TypeScript + Hero UI + Tailwind CSS + Lucide React

项目结构：全部按类型组织 (components、hooks、lib、types 等)，不按功能模块组织。

项目初始化命令：

# Web 端（Next.js + HeroUI + Tailwind + Biome）
bunx create-next-app@latest my-web \
  --typescript \
  --tailwind \
  --biome \
  --app \
  --use-bun \
  --yes

# 移动端（React Native + Expo + NativeWind + HeroUI Native）
npx create-expo-app@latest my-mobile --template blank
cd my-mobile
bun add nativewind@preview lucide-react react-native-reanimated react-native-safe-area-context
bun add -D tailwindcss @tailwindcss/postcss postcss

# 浏览器插件端（WXT + React + HeroUI + Tailwind）
bunx wxt@latest init my-extension
cd my-extension
bun add @heroui/react@latest lucide-react
bun add -D tailwindcss

# 注意事项：
HeroUI v3 必须搭配 Tailwind CSS v4，禁止降级。
样式优先使用 HeroUI 组件 + Tailwind 类，尽量不写原生 CSS。
采用 HeroUI v3 新组件名与 Compound Components 写法。
globals.css 必须包含：css

@import "tailwindcss";
@import "@heroui/styles";

所有命令使用 Bun 执行。
代码必须通过 Biome 格式化。
TypeScript 严格类型，优先用 Zod 推导。
网页Web端与浏览器插件保持 HeroUI v3 + Tailwind v4 一致风格。
新组件默认支持 asChild 和 className 透传。
优先使用 Server Components，能 Server 就不用 Client。
