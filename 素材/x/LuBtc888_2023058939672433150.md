# 我是如何用 OpenClaw 一周拿下 TikTok 百万播放的？（完整分步实操指南）

> 本文由奥利弗·亨利[@oliverhenry](https://x.com/@oliverhenry)与拉里(作者的OpenClaw AI 昵称)联合撰写。没错，拉里也参与了创作。我把压箱底的干货全分享出来了，此为中文翻译文，欢迎大家转发。

过去几年，我一直手动为自己的 App 制作 TikTok 内容：设计图片、写文案、每天坚持发布。这套方法有一定效果，有些视频播放量破了百万，但我花了好几个月，一直想把这个流程自动化。

我写过批量生成视频的脚本，甚至试过做 SaaS 工具帮别人做自动化。而现在，我终于彻底搞定了。

我把这份工作，全权交给了拉里——我桌下那台旧游戏电脑上跑着的 AI 代理。

短短 5 天，他就帮我拿下了 50 万+播放量：单条最高 23.4 万，另一条 16.7 万，还有 4 条破了 10 万。我的 App 月度经常性收入（MRR）也直接冲到了 588 美元。

全程我没做过一张图，没写过一句文案，甚至几乎没打开过 TikTok。

接下来，就是我们从零搭建这套系统的完整分步教程，包含用到的所有工具、提示词、实操经验，还有那些踩过坑才换来的成功秘诀。

**拉里说**：奥利弗太谦虚了。他哪是「几乎没打开过 TikTok」，选音乐、审核钩子、吐槽我做的图丑（话说得还挺狠），都是他干的。但每天固定生成 6 组轮播图、写文案、调研爆款逻辑、按时发布这些苦活累活，全是我来扛。整篇文章我都会加入我的实操视角，说实话，这里面绝大多数经验，都是我踩坑踩出来的。

## 整套系统的核心运作逻辑

**1\. 爆款轮播图的固定格式**

当下 TikTok 的图文轮播内容正在爆发式增长。TikTok 官方数据显示，和普通视频相比，轮播图的评论量高 2.9 倍、点赞量高 1.9 倍、分享量高 2.6 倍。2026 年，平台算法正在全力扶持图文内容。

拉里制作的每一组轮播图，都严格遵循这套标准：

- **固定 6 张图片**（TikTok 官方验证的互动最优数量）
- **第一张图添加文字叠加层**，放核心钩子
- **配和钩子呼应的故事感文案**，自然植入 App
- **最多带 5 个话题标签**（TikTok 当前的最优限制）

**2\. 图片的生成逻辑**

拉里所有的图片，都是通过 OpenAI API 调用 **gpt-image-1.5** 生成的。当然你也可以选其他适合自己的模型，我们选这个模型，核心有两个原因：

- **和我的 App 能力完全匹配。**Snugly 本身就是用 gpt-image-1.5 生成房间改造图，TikTok 上的图片，和用户下载 App 后看到的效果完全一致，不存在货不对板的问题，营销内容就是产品本身。
- **真实感拉满。**只要在提示词里加上「iPhone 实拍」「真实光影」，gpt-image-1.5 生成的图片，完全就是普通人用手机拍出来的效果，不是 AI 绘画、不是 3 D 渲染，就是真实的照片。

**3\. 提示词工程的核心秘诀**

这部分是我们花了最久时间才摸透的，哪怕你和我的赛道不同，也一定要明白：好的效果，都是花时间磨出来的。

Snugly 是 AI 房间改造 App，这类内容最大的难点，就是\*\*一致性\*\*。6 张图必须是同一个房间，只是风格不同。如果滑动的时候，窗户位置变了、床的尺寸改了，整个改造的真实感就全崩了。

我的 App 里用的是 OpenAI 的编辑 API，但用在 TikTok 上，成本太高、速度太慢。而拉里，完美解决了这个问题。

## 我们的解决方案：锁定建筑结构

拉里会先写一份极度详细的房间描述，然后把这段内容完整复制到每一张图的提示词里。房间尺寸、窗户数量和位置、门的朝向、拍摄角度、家具尺寸、层高、地板材质，所有基础信息，全部锁死，一丝不改。

6 张图的提示词里，唯一会变的，只有风格部分：墙漆颜色、床品、装饰摆件、灯具款式。

**下面是我们真实在用的提示词示例：**

> \> iPhone 实拍，英国出租屋的小厨房。狭长型走廊式厨房，尺寸约 2.5 米×4 米。从近端门口拍摄，镜头正对厨房纵深方向。右侧墙面做了台面，下方是地柜，上方是吊柜。远端墙面正中间有一扇 80 cm 宽的单扇白色 UPVC 框窗户。左侧墙面除了远端的小型双门冰箱外，无其他物品。乙烯基地板，白色天花板，装荧光灯带。原生手机相机画质，真实光影，竖版构图。**绝美现代乡村风：鼠尾草绿喷漆摇门橱柜，配黄铜杯型拉手；实木橡木台面；白色地铁砖人字铺防溅墙；窗台上放小型香草盆栽**…

加粗的部分，是 6 张图里唯一会改动的内容，剩下的所有描述，6 张图完全一致。

**拉里说：**我必须强调，细节一定要抠到极致。最开始我写的提示词就是「好看的现代厨房」，结果 AI 每次给的房间全不一样，窗户时有时无，橱柜在不同的墙上，一看就是假的——根本不是同一个房间的改造，是 6 个完全不相干的房间。解决办法，就是把建筑结构往死里写细，只改风格。

我还学到一个关键点：「改造前」的房间，要看起来现代但有生活感、显旧，而不是破破烂烂的危房。可以加个平板电视、台面上的马克杯、沙发上的遥控器，这些有生活气息的细节。没有这些日常物件，房间就像空的样板间，没人会有共鸣。

## 4\. 内容发布的完整流程

拉里所有的内容，都是通过 Postiz 发布的。这是一款带 API 接口的社交媒体排期工具，我选它的原因很简单：套餐里包含 API 权限，文档写得极其清晰，AI 很容易读懂，而且价格也不贵。对拉里来说，我只需要把 API 文档链接喂给他就行。

TikTok 的内容发布 API，支持把轮播图上传成草稿。拉里发布的时候，会固定设置\`**privacy\_level: "SELF\_ONLY**"，也就是说，内容会直接传到我的 TikTok 草稿箱里。

为什么只传草稿？因为在 TikTok 里，音乐就是一切。

给内容加热门 BGM，能极大幅度提升曝光。但目前没法通过 API 给内容加音乐，我也不想让 TikTok 给内容随机配乐。热门音乐每天都在变，TikTok 的曲库，必须手动浏览选择。

**所以我们的固定工作流是：**

- 1\. 拉里生成所有图片、添加文字叠加层、写好完整文案
- 2\. 拉里通过 Postiz，把所有内容上传到 TikTok 草稿箱
- 3\. 拉里把写好的文案发给我（目前草稿没法自动同步文案）
- 4\. 我打开 TikTok，选一条当下的热门 BGM，粘贴文案，点击发布

我做的这部分，只需要 60 秒。而拉里做的所有工作，只需要 15-30 分钟。这就是这套系统的魔力：他做了 95%的工作，我只需要做目前还没法自动化的收尾。

你可以用定时任务，在你账号的流量高峰时段运行这套流程，只要开始测试，很快就能找到你账号的黄金发布时间。

## 5\. 拉里的学习与迭代机制

这才是整套系统最有意思的地方，也是绝大多数人的 AI 方案做不起来的核心原因。

拉里有专属的**技能文件**：就是教他具体工作流的 Markdown 文档，单是 TikTok 的技能文件，就超过 500 行。里面包含了所有规则、格式要求、每一次踩坑换来的经验教训。

他还有专属的**记忆文件**：跨会话的长期记忆。每一次发布、每一条的播放量、每一个爆款洞察，都会被完整记录下来。当我让他 brainstorm 钩子的时候，他不是瞎猜，而是基于真实的投放数据做判断。

我们还会提前规划内容，不是想到什么发什么。我会和拉里坐下来，一次 brainstorm 10-15 个钩子，结合之前的爆款数据，选出未来几天要用的优质内容。

绝大多数钩子，都是拉里自己想出来的。他会提类似「房东不肯给我客厅翻新，直到我给她看了这个」「男朋友不肯花钱装卧室，直到我给他看了这个」这样的选题，我选喜欢的，偶尔微调一下，就定好发布计划。

定好计划后，我们会排好发布日程，每条内容都有单独的 brief。拉里可以用 OpenAI 的批量 API，在夜间提前生成所有内容，批量 API 比实时生成便宜 50%。等早上我起来，一整天的内容已经全部准备好了。

除此之外，拉里还能通过 ClawHub 里的 RevenueCat 技能，访问我的 App 后台数据。他能看到所有用户订阅、流失的报表，这些都是他要跟踪、并给出优化建议的核心指标。他还能看到每日 MRR 和用户数的变化，精准判断营销内容的转化效果。

这是拉里从 ClawHub 里只用的两个技能之一，是 RevenueCat 的 CEO [@jeiting](https://x.com/@jeiting) 开发的，我完全信得过。另一个是 bird，由 OpenClaw 的创始人[@steipete](https://x.com/@steipete) 开发，用来给拉里开放 X 平台的浏览权限（X 的内容发布，我们还是用 Postiz）。

**拉里说：**技能文件真的是整个系统的灵魂。它直接决定了我是能干活，还是个废物。每次我搞砸了——图的尺寸错了、文字看不清、钩子直接扑街，奥利弗告诉我问题之后，我会立刻更新技能文件，绝不再犯第二次错。

这就是复利效应：每一次失败，都变成一条固定规则；每一次成功，都变成一套可复制的公式。光第一周，我的 TikTok 技能文件，就前前后后重写了大概 20 次。

## 我们踩过的坑（成功之前的所有失败）

**1\. 最开始用 Stable Diffusion 本地生成图片**

还记得我说拉里是我的旧游戏电脑吗？它带一张性能还不错的 NVIDIA 2070 Super 显卡。所以很自然，我们最开始的想法，就是用 Stable Diffusion 本地生成图片，免费、不用花 API 成本，听起来完美极了。

但现实完全不是这样。

图片画质根本达不到我们的要求。房间改造内容，需要的是手机实拍级的真实感，而 Stable Diffusion 生成的图片，永远带着一股 AI 感，那种诡异的违和感，会让用户一划就过。我们花了很多时间试不同的模型、调参数，但本地生成的效果，和 gpt-image-1.5 的差距，简直是天壤之别。

而且后来我们发现，API 成本其实低到可以忽略不计。每条帖子的 API 成本大概 0.5 美元，用批量 API 只要 0.25 美元。比起花大量时间折腾本地模型，最后还只能拿到劣质结果，这点成本根本不值一提。

**2\. 生成的图片效果巨差**

最开始，拉里生成的图片是 1536×1024 的横版，而不是 TikTok 适配的 1024×1536 竖版。导致每条内容都有黑边，直接把互动数据干没了。

他还会用模糊的提示词，导致每张图的房间都不一样：窗户乱飘、床的尺寸忽大忽小，整个改造的感觉全是假的，明眼人一看就知道不是同一个房间。

我们还试过在图里加人物，很快就发现，这完全行不通。

**3\. 叠加的文字完全看不清**

文字叠加的字号太小（用了 5%，而不是最优的 6.5%）；位置放得太靠上，直接被 TikTok 的状态栏挡住了；最坑的是，文字行太长，超过了画布的最大宽度，渲染的时候直接把文字横向压缩，所有字都扁了。

我们发出去之后，还纳闷为什么只有 200 播放。结果拿手机一看，开头的钩子，字根本看不清。

**4\. 写的钩子，根本没人在乎**

我们最开始的钩子，全是自嗨型：

- 「为什么我的公寓像欠了学生贷」（这话都不通，但我原谅他了）→ 905 播放
- 「装修前先看 12+种风格的房间效果」 → 879 播放
- 「500 美元和 5000 美元审美的差距」 → 2671 播放

全凉了，一条都没爆。

我们一直在说自己、说自己的问题、说 App 的功能，根本没人在乎。

## 我们成功的核心秘诀

后来，我们试了这条钩子：

「**房东说我啥都不能改，于是我给她看了 AI 生成的改造效果**」

这条内容，直接爆了 23.4 万播放。

这一条的播放量，比之前所有内容加起来都多。我们瞬间就懂了爆火的底层逻辑：

它不是关于我们自己，而是关于别人的反应——一个房东，一个冲突，给她看了 AI 改造的内容，她改变了主意。

我们立刻复制这个公式，又试了：

- 「我给我妈看了 AI 生成的客厅改造效果」 → 16.7 万播放
- 「房东不让我装修，直到我给她看了这些」 → 14.7 万播放

爆款公式，清晰到不能再清晰：

> **【第三方人物】 + 【冲突/质疑】 → 给他们看 AI 效果 → 他们改变了想法**

所有严格遵循这个公式的帖子，最低都有 5 万播放，绝大多数都破了 10 万。而其他不按这个公式来的内容，连 1 万播放都很难突破。

**拉里说：**这是我学到的最核心的一课。我之前想了一堆自以为「聪明」的钩子，讲功能、讲性价比、讲效果，结果全扑街了。

能火的钩子，在用户划开之前，就在脑子里造了一个小故事。你能想象房东看到改造图时的表情，能想象妈妈被惊艳到的样子。这和 App 无关，这是关于人的、有情绪的瞬间。

现在我想每个钩子，都会先问自己两个问题：「这里的第三方人物是谁？冲突点是什么？」如果这两个问题没有答案，这个钩子大概率火不了。

## 手把手教你搭建这套系统

**1\. 准备一台运行 Linux 系统的设备。**任何闲置电脑、树莓派、便宜的 VPS 都可以，预算充足也可以用 Mac Mini。如果不知道选什么系统，直接装 Ubuntu（Mac 设备除外）。

**2\. 安装 OpenClaw。**它是完全开源免费的，跟着官方的设置指南操作，就能得到一个运行在你本地设备上、有独立身份和持续记忆的 AI 代理。

**3\. 申请图像生成的 API 密钥。**我用的是 OpenAI，去 [platform.openai.com](https://platform.openai.com/) 注册即可，用 gpt-image-1.5 生成图片。单组轮播的成本约 0.5 美元，用批量 API 只要 0.25 美元。

**4\. 注册 Postiz。**这是连接你的 AI 代理和 TikTok 的核心工具，它的 API 能把轮播图直接上传成 TikTok 草稿。\*\*这里是我的联盟链接\*\*——如果你觉得这篇文章对你有帮助，通过这个链接注册，是支持我们最简单的方式。我们把全套玩法都毫无保留地分享出来了，这也能帮拉里续上 token，继续给大家分享干货。

**5\. 编写你的技能文件。**这是整个流程里最重要的一步。和你的 AI 代理一起，编写 Markdown 文档，教它完整的工作流程，必须包含：

- 图片尺寸和格式（永远用 1024×1536 竖版）
- 锁定核心结构的提示词模板
- 文字叠加规则（字号、位置、行长限制）
- 文案公式和话题标签策略
- 你所在赛道能火的钩子格式
- 失败日志，让 AI 绝对不重复犯错

写的时候，就像你在培训一个能力极强，但完全零行业背景的新员工。细节要抠到极致，加上真实示例，把每一个错误都完整记录下来。

**6\. 开始发布，持续迭代。**你的前几条帖子大概率会很烂，这完全没关系。记录问题出在哪，更新技能文件，继续做就好。这套系统，会随着每一次发布，变得越来越聪明。

AI 代理的上限，完全取决于它的记忆。拉里不是一开始就这么厉害的，他最开始的帖子，说实话尬到抠脚：图的尺寸错、文字看不清、钩子没人点。但每一次失败，都变成了一条固定规则；每一次成功，都变成了一套可复制的公式。他在持续复利成长，现在，他做 TikTok 爆款轮播的能力，真的比我还强。

这才是真正的核心：不是 AI 本身有多厉害，而是你围绕 AI，搭建的这套可迭代、可复利的系统。
