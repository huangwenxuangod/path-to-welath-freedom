接下来我们来讲重头戏，如何使用 Skill 来写小说！

在claude 网页端虽然也可以写小说，但也是会因为写的多了，出现续写困难，或者是前文太长导致幻觉。使用 Skill 就不一样了， 在规则里只要写清楚，以后每次写的时候一定要回顾，并且写到那里还要给用户以提示。

这样，以前的大纲，人物小传，目录，就会在同一个文件夹中，再写的话就直接调用本地内容，而不再一次浪费 token重新写了。

我来教大家如何操作，今天不教如何制作这样的 Skill 那内容太长了，以后有机会我再来讲解。

这里还要讲一个大家容易忽视的设定，就是 /.claude 的文件夹，其实才是程序的全局目录，在这个目录里的 Skills 文件夹内容，就是你在日常工作中，可以随时调用的。

如果，你在下载文件夹中，又新建了一个 claude code 的文件夹，这里也新建了一个 skills，或projects文件夹，可以吗？

当然可以，这样更有利于管理日常的项目和某个项目的 skills 。

那这里全局的 skills 和我私用的 skills 有什么不同？

一句话先给结论：

全局 skills = 操作系统级能力库

私用 skills = 项目级战术工具

存放位置：

全局 Skills存放位置 ~/.claude/skills/

私用 Skills 存放在某个项目目录内，如 projects/test/skills/ ，或新建目录下 skills

作用范围：

全局 Skills:所有 Claude Code 会话、所有项目,设计目标长期复用、跨项目都可以。

私用 Skills:仅当前项目,针对性解决当前问题。“这是这个项目的特殊需求”才使用。

根据上面的说明，我们的小说 Skills 应该放到全局 skills 里，因为这是我们长期会反复使用的技能。

你以后每遇到一个 skill，都问自己一句话：

“我换一个项目 / 换一本小说，还会不会用它？”

1.  会 → 全局 ~/.claude/skills

2.  不会 → 项目 projects/xxx/skills


其实这里还有个重要知识点，叫 CC 的行为规范，一般用 CLAUDE.md来书写，它就像是个总的说明规则，但我这里先不做详细介绍，不影响你现有操作，后期我们skills 多了再说。

CLAUDE.md 的本质是：作用域规则文件，不是 skill 文件！

描述的结构大概是这样的：

```
my-novel/
├── .claude/
│   └── CLAUDE.md      ← 项目规则
├── skills/
│   ├── novel/
│   │   ├── workflow.md
│   │   ├── plot.md
│   │   └── style.md
├── chapters/
├── outline.md
```

CLAUDE.md 是：告诉 Claude Code，在“这个目录及其子目录里，你要长期遵守哪些规则”。

Skill 是什么时候生效？这很多人会误解

skill不会因为文件存在就自动生效，而需要你显式声明如：

```
请使用 /skills/writing/style.md 中定义的写作风格。
```

又或者笨一点说：帮我搜索一下 skills 里写小说的 skill 调用，我要写小说。都行，AI 是很聪明的！

即使你不说出 skill 的全英文名，大概记着是什么功能就行。调用，操作，完成！

现在你不知道有多少 skills ，就输入"/skills" 然后下面就出来了。

还有种情况，是你在操作过程中，又有了新想法，要去做别的事情，你可以新开终端，当然也可以在这个窗口里继续，但如果太长的话，需要压缩上下文让你等很久，我个人建议是一个任务开一个终端窗口，别在一个窗口里干太久。

结论一句话：

Claude Code 只有在你“明确要求写入文件”时，才会把内容落到当前项目目录。

否则，它只是在“聊天里生成文本”，不会自动保存到任何文件夹。

这不是 BUG，CC生成≠写入文件

你只有明确告诉他，把写作的内容放到 tset 文件夹中，才会正常写入！

那现在怎么办？我当时也不知道，反正不知道就试呗，我们都是菜鸟谁怕谁？
